# Admin Document View Implementation

## ✅ What Has Been Added

### 1. View Documents Button in Employee List
- Added **purple "View Documents" button** (Eye icon) in the employee actions column
- Button appears as the first action for each employee in the list
- Opens a modal showing all documents for the selected employee

### 2. Document Viewing Modal
**Modal Features:**
- Full-screen modal with employee name and employee number in header
- Shows total document count
- Lists both:
  - **Employee Uploaded Documents** - Documents uploaded by the employee
  - **Admin Generated Documents** - Documents generated by admin (offer letters, certificates, etc.)

**Document Information Displayed:**
- Document name/type
- Document description (if any)
- Verification status with color-coded badges:
  - 🟢 Green = Verified
  - 🟡 Yellow = Pending
  - 🔴 Red = Rejected
- Upload date
- File size (for uploaded documents)

### 3. Handler Functions Added

#### `handleViewEmployeeDocuments(employee)`
- Fetches both employee-uploaded and admin-generated documents
- Opens modal with loading state
- Combines both document types in a single view

#### `handleDownloadEmployeeDocument(doc)`
- Downloads employee-uploaded documents from storage
- Opens admin-generated documents (PDFs) in new tab
- Handles both document types intelligently

#### `handleVerifyDocument(docId, status, comments)`
- Admin can verify or reject employee documents
- Updates verification status in database
- Reloads document list after verification
- Supports optional comments for rejections

### 4. Document Types Shown

**Employee Uploaded:**
- Aadhar Card, PAN Card, Passport
- Driving License, Voter ID
- Education Certificates
- Experience Letters
- Bank Proof, Medical Certificates
- Resume/CV, Photos
- Other documents

**Admin Generated:**
- Offer Letters
- Salary Certificates
- Form 16, Form 24Q
- Experience Certificates
- Relieving Letters
- Other employment documents

## 🎯 How It Works

### For Admin:
1. Go to **HR & Employment Documents** section
2. In the **Employees** tab, find any employee
3. Click the **purple Eye icon** (first button in Actions column)
4. Modal opens showing all documents for that employee
5. View document details including verification status
6. Click **Download** button to download any document
7. Click **Close** to exit modal

### Document States:
- **Pending** - Yellow badge, awaiting verification
- **Verified** - Green badge, admin approved
- **Rejected** - Red badge, admin rejected with comments

## 🔧 Technical Details

### State Variables:
```typescript
const [showEmployeeDocsModal, setShowEmployeeDocsModal] = useState(false);
const [employeeDocuments, setEmployeeDocuments] = useState<any[]>([]);
const [selectedEmployeeForDocs, setSelectedEmployeeForDocs] = useState<Employee | null>(null);
const [loadingEmployeeDocs, setLoadingEmployeeDocs] = useState(false);
```

### Data Flow:
1. Click "View Documents" button → `handleViewEmployeeDocuments(employee)`
2. Fetch documents from:
   - `employeeDocumentService.getDocuments({ employee_id })` - Uploaded docs
   - `employeeService.getEmploymentDocuments(employee_id)` - Generated docs
3. Combine both arrays and display in modal
4. Each document shows appropriate actions based on type

### Document Detection:
- **Employee Uploaded**: Has `storage_path` property
- **Admin Generated**: Has `pdf_url` property (no `storage_path`)
